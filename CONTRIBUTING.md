# Contributing to The AI Memory Manifesto

Thank you for your interest in contributing. This is an open document — we welcome input from builders, researchers, designers, lawyers, policymakers, and users.

## How to contribute

### Feedback and discussion

- **Open an issue** to propose changes, flag problems, or start a discussion.
- Use descriptive titles that reference the specific section or principle.

### Suggesting edits

1. Fork this repository.
2. Create a branch (`git checkout -b your-change`).
3. Make your edits.
4. Submit a pull request with a clear description of what you changed and why.

### What kinds of contributions are welcome

- **Language improvements** — clarity, precision, tone.
- **New arguments or evidence** — if a claim needs stronger grounding, help us strengthen it. See [Evidence](docs/evidence.md) for documented provider practices. All factual claims in the evidence document must link to a primary source (official announcement, policy page, regulatory decision, or peer-reviewed publication). See the [Citation Standard](docs/evidence.md#citation-standard) section for formatting rules.
- **Screenshots** — UI settings, default states, and policy pages change over time. Dated screenshots in [`docs/screenshots/`](docs/screenshots/) preserve the evidence. Name files as `YYYY-MM-DD-provider-description.png` and reference them inline in the evidence document.
- **Translations** — help make the manifesto accessible in other languages.
- **Technical contributions** — schemas, reference implementations, format proposals that advance the technical agenda.
- **Legal and policy analysis** — mapping the manifesto's demands to existing legal frameworks.
- **Related work** — pointing us to initiatives, papers, or projects we should acknowledge. See [Landscape](docs/landscape.md) for what we already track.
- **Terminology** — improving definitions or adding missing terms. See [Terminology](docs/terminology.md) for the shared vocabulary.

### What this is not

This is a manifesto — a declaration of principles. Contributions should strengthen the argument, not dilute it. We are not looking for:

- Additions that weaken the core claims in favor of vendor convenience.
- Marketing language or promotional content.
- Changes that remove rights or weaken protections.

## Code of conduct

Be respectful. Argue in good faith. Remember that this document exists to protect people.
